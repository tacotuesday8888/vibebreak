# Contributing to Vibebreak

Thanks for helping improve Vibebreak.

## Before you start

- Keep changes small and easy to review.
- Keep the project beginner-friendly.
- Do not add backend services, auth, cloud services, AI, payments, analytics, or unnecessary complexity.
- Open an issue first for larger gameplay or project structure changes.

## Local setup

Install dependencies:

```bash
npm install
```

Run the game:

```bash
npm start
```

Run checks before opening a pull request:

```bash
npm run typecheck
npm run build
```

## Pull requests

When opening a pull request:

- Explain what changed.
- Mention any issue it fixes.
- Include manual test notes when behavior changes.
- Keep unrelated cleanup out of the same pull request.

## Reporting bugs

Please include:

- What happened
- What you expected
- Your operating system
- Your Node.js version
- Steps to reproduce the issue
